Features

What Agentklar does.

It makes AI-assisted coding reliable: it tracks the work, runs your real checks, keeps the proof, and only lets a human sign off. Here's everything, in the order a task moves through it.

The workflow

Every task walks the same path.

State lives in a local database, not in chat — so any session picks up where the last left off.

draft→ ready→ in progress→ review→ auto qa→ you approve→ done

review + qa are automatic gates · you approve is human-only · done is final

Evidence

Only proof Agentklar ran itself counts.

Machine-attestedAgentklar ran it. Command, exit code, timestamps, and a hash of the log. The only evidence that satisfies a required check.
Agent-reportedA model claimed it. Kept as support, marked untrusted until reproduced or accepted.
Human-observedYou checked it. A manual observation, attributed to your account.

Verification levels

Your commands, grouped.

Agentklar runs the checks your project already declares. It never bundles a test framework or invents a command.

L0Inspectformat, lint, types
L1Unitfocused unit & component tests
L2Integrationservice, database, migration
L3Systemend-to-end, UI, performance

Community library

Build a feature. Publish it with git push.

No marketplace, ever. A community feature is just a git repo with a manifest — installed and pinned by package tooling you already know. Four kinds: review lenses, quality recipes, project skills, agent configs.

pack initscaffold from a reviewed template
pack validatecheck the manifest, format, and permissions
pack testrun against fixtures — catch defects, spare clean code
pack publishtag in git, optionally PR the community index
# install — resolves, pins, shows cost before enabling
agentklar pack add github.com/example/django-review
agentklar pack enable django-review --project

Nothing new to run. It reuses git, APM, the Agent Skills format, GitHub PRs, and a curated index repo — enabled explicitly, cost shown first, pinned and hashed.
